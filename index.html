Hivemind

One agent template, every use case — coding, searching, and code review — with the internet on demand.

Multi-agent swarm Live web search FastAPI microservices Ollama · Groq

▍The idea

Hivemind is a universal swarm-agent service. Every role — researcher, coder, reviewer, generalist — is the same reusable agent (just instructions + tools + handoffs). A triage agent reads each request and routes it to the right specialist, and any agent can connect to the internet the moment a task needs fresh information. It's all exposed as clean, microservice-style FastAPI endpoints.

▍How it works

🧭 triage
classifies & routes
→
🔎 researcher
searches the web, cites sources
💻 coder
writes & fixes code
🔍 reviewer
reviews code, asks first
💬 generalist
everything else

Agents cooperate through handoffs: one request can flow triage → coder → reviewer, each building on the last. When the task looks like it needs the web ("search…", "latest…", a URL), the search tools switch on automatically.

▍What it can do

💬

Chat that searches

Talk to it normally; when a question needs current facts, it goes online by itself and shows a live "Searching the web…" loader.

🌐

Key-free web access

Searches via a self-hosted SearXNG (Docker) with a DuckDuckGo fallback — then reads the pages and synthesizes an answer.

🧩

Pluggable models

Ollama for fully local, or Groq for fast cloud — behind one interface, switchable per request.

🔍

Smart code review

Point it at a file; the reviewer asks clarifying questions first, then reviews with your answers in mind.

📡

Streaming events

A Server-Sent-Events endpoint streams every step (routing, tool calls) — ready to power a live UI.

🧠

Conversation memory

Sessions keep context across messages, so follow-ups just work.

▍Try it

curl -X POST http://127.0.0.1:8000/swarm/run \
  -H "Content-Type: application/json" \
  -d '{"task": "search the web for the latest Python version"}'

The triage agent routes it to the researcher, which searches, reads a page, and answers with sources. Or just chat: python chat.py.
